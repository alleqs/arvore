1. ao fim de cada apresentação, árvore com documento base apropriado é gerada, serializada e salva na pasta criada para aquele dia
2. diariamente (à 0:30, por exemplo) nova rotina é executada para ler todos os arquivos da pasta do dia anterior, desserializá-los e apensá-los numa única lista. Feito isso, a lista é serializada e salva
3. assim, toda vez que se rodar a rotina para montagem das tabelas do big data, o arquivo com a lista de documentos base do item "2" é usado para se recuperar o(s) CTe(s) e MDFe (quando aplicável) vinculados a ele
